Key Data Flows

Clock In Flow

  1. Employee opens app → biometric prompt
  2. Fingerprint verified locally
  3. App reads GPS coordinates
  4. Checks for mock/fake location
  5. Sends to API: lat, lng, device_id, token
  6. Server validates: JWT → device → geofence
  7. Server records attendance in MySQL
  8. Returns success/error to app

Leave Application Flow

  1. Employee selects leave type & dates
  2. App checks remaining balance locally
  3. Sends leave request to API
  4. Server validates balance & conflicts
  5. Stores as "pending" in MySQL
  6. Push notification to admin
  7. Admin approves/rejects from dashboard
  8. Push notification back to employee

Payroll Calculation

  1. Admin triggers monthly payroll
  2. System counts present days per employee
  3. Deducts approved leaves from balance
  4. Calculates: (monthly salary / total days) × present days
  5. Deducts LWP (unpaid leave) days
  6. Generates salary slip PDF
  7. Employee views slip in app

Device Binding Flow

  1. Admin creates employee (username + password)
  2. Employee installs app, enters credentials
  3. First login: app registers device ID with server
  4. Server binds employee → device_id
  5. Subsequent logins: server verifies device match
  6. New device? Admin must reset binding
  7. Prevents buddy-punching across devices

Tech Stack Summary: Flutter 3.27+ (Riverpod) → PHP REST API (Vanilla, JWT) → MySQL | Firebase FCM for notifications | Google Maps API (admin only) | Geolocator + local_auth + persistent_device_id (Flutter)

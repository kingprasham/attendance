Critical Endpoint Details

POST /attendance/clock_in

Request:
{ latitude, longitude, device_id }
Header: Authorization: Bearer <jwt>

Server Validates:
1. JWT token valid → extract employee_id
2. device_id matches bound device
3. No existing clock_in today
4. Haversine(employee_lat/lng, branch_lat/lng) ≤ radius
5. Check for fake GPS flag from client

Response:
{ success: true, clock_in_time: "09:05 IST" }
{ success: false, error: "Not in office premises" }

POST /auth/login

Request:
{ username, password, device_id }

First Login (no bound device):
1. Verify credentials
2. Bind device_id to employee
3. Return JWT access + refresh tokens
4. Enable biometric prompt on client

Subsequent Logins:
1. Biometric verified on client
2. Send stored credentials + device_id
3. Server checks device match
4. Return fresh JWT tokens

Wrong device? → "Device not registered"

POST /salary/generate

Request (admin only):
{ branch_id, month, year }

Calculation Logic:
1. Get all active employees in branch
2. For each employee:
  • Count present days from attendance_logs
  • Count approved leave days
  • Count LWP (unpaid) days
  • per_day = monthly_salary / total_working_days
  • net = per_day × (present + paid_leave) days
3. Generate salary_slips records
4. Push notification to employees

Authentication & Security

JWT Strategy:
• Access token: 30 min expiry
• Refresh token: 30 days expiry
• Refresh rotation (new refresh each use)

Rate Limiting:
• Login: 5 attempts / 15 min
• Attendance: 10 requests / min
• General API: 100 requests / min

Data Security:
• Passwords: bcrypt hash
• PAN/Aadhar: AES-256 encrypted
• All inputs: sanitized + validated
• PDO prepared statements (no SQL injection)
Total: ~30 endpoints — 8 auth/device, 4 branches, 7 employees, 5 attendance, 7 leaves, 3 salary, 3 holidays, 2 notifications, 2 leave policies. All behind JWT auth. Admin endpoints require admin token.
